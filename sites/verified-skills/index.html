Verification note / methodology

Trust is a trail,
not a badge.

A scan can reduce risk; it cannot make an instruction package harmless. Verified Skills keeps the evidence close to the decision and preserves the exact version you reviewed.

  1. 01
    Read the behavior

    Inspect instructions, scripts, external access, and requested tools.

  2. 02
    Pin the source

    Install the reviewed commit—not a moving branch or mutable download.

  3. 03
    Test in isolation

    Try a representative prompt before enabling the skill in daily work.

  4. 04
    Remove cleanly

    Keep a file inventory so uninstall touches only what the skill added.
